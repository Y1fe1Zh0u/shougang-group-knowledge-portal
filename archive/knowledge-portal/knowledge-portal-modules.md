## 3. 接口清单

### 3.1 新增接口

| 接口 | 路径 | 说明 |
| --- | --- | --- |
| 文件列表 | `GET /api/v1/knowledge/space/{space_id}/files` | 按空间查文件，支持筛选 |
| 跨空间标签查询 | `GET /api/v1/knowledge/files` | 跨空间按标签查文件，服务首页分区和通用列表页 |

### 3.2 复用现有接口

| 接口 | 路径 | 说明 |
| --- | --- | --- |
| 空间标签列表 | `GET /api/v1/knowledge/space/{space_id}/tag` | BiSheng 已有，获取空间下全部标签 |
| 技术问答 | `POST /api/v1/workstation/chat/completions` | BiSheng 日常模式，SSE 流式返回 |

## 4. 公共数据结构

### 4.1 `KnowledgeFileItem`（列表项）

```json
{
  "id": 1580,
  "title": "热轧1580产线精轧机振动纹治理实践",
  "summary": "针对 1580 热连轧精轧机组出现的周期性振动纹缺陷...",
  "source": "轧线技术案例库",
  "updated_at": "2026-04-13T10:30:00",
  "tags": ["热轧", "精轧机", "振动纹"]
}
```

| 字段 | 类型 | 来源 | 规则 |
| --- | --- | --- | --- |
| `id` | `int` | `KnowledgeFile.id` | 直接取 |
| `title` | `string` | `KnowledgeFile.file_name` | 去掉文件后缀 |
| `summary` | `string` | `KnowledgeFile.abstract` | 空值返回 `""` |
| `source` | `string` | `Knowledge.name` | 关联查询 |
| `updated_at` | `datetime` | `KnowledgeFile.update_time` | 直接取原始值 |
| `tags` | `string[]` | `Tag.name` via `TagLink` | 空时返回 `[]` |

### 4.2 `KnowledgeFileDetail`（详情）

继承 `KnowledgeFileItem` 全部字段，额外增加：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `space` | `{id: int, name: string}` | 所属空间信息 |

不返回 `content` 正文字段。文件正文通过已有的预览接口获取预览 URL，前端渲染展示。

## 5. 通用约定

- 统一响应包装：`{ "code": 0, "message": "success", "data": {} }`
- 分页：`page` 默认 1，`page_size` 默认 20，最大 100
- `id` 类型：`int`
- 时间字段：`datetime` 原始值，FastAPI 自动序列化
- 权限：第一版不做鉴权，只查公开数据（`is_released=True` / `auth_type=public`）
- 前端维护 `domain → space_id` 映射（一对一关系），后端只接收 `space_id`

## 6. 模块一：全局搜索

### 6.1 功能描述

首页顶部搜索框，用户输入关键词，跨所有公开知识空间搜索文件内容。

### 6.2 前端职责

- 搜索框 UI 和交互
- 热门搜索词（前端写死）
- 搜索结果跳转到搜索结果页

### 6.3 后端职责

- 跨空间全文搜索（标题 + 摘要 + 正文）
- 搜索基于 BiSheng 已有的 Elasticsearch 索引，正文在文件上传时已建好索引
- 需要新包装一个跨空间搜索服务，现有 `search_space_children` 只支持单空间

### 6.4 接口

```
GET /api/v1/knowledge/search?q=振动纹&space_id=12&page=1&page_size=20
```

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `q` | `string` | 是 | 搜索关键词 |
| `space_id` | `int` | 否 | 限定单个空间，不传则搜全部公开空间 |
| `page` | `int` | 否 | 默认 1 |
| `page_size` | `int` | 否 | 默认 20，最大 100 |

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [KnowledgeFileItem],
    "total": 23,
    "page": 1,
    "page_size": 20
  }
}
```

### 6.5 搜索实现方案

1. 查出所有公开空间的 ES 索引名（`Knowledge.index_name`，`is_released=True`）
2. 如果传了 `space_id`，只搜该空间的 ES 索引
3. 如果没传，用 `MultiRetriever` 搜所有公开空间的 ES 索引
4. ES 返回匹配的文件 ID
5. 用文件 ID 查 SQL 拿 `KnowledgeFile` 元数据
6. 组装为 `KnowledgeFileItem` 返回

## 7. 模块二：搜索结果页

### 7.1 功能描述

展示搜索结果列表，支持筛选、排序、分页。顶部展示 AI Overview（基于日常模式 API 的 AI 总结）。

### 7.2 前端职责

- AI Overview 区域（SSE 流式渲染）
- 筛选栏（业务域、文档类型、标签）
- 排序切换（相关性优先 / 最近更新优先）
- 搜索结果列表渲染
- 分页控件
- 点击结果跳转到详情页

### 7.3 后端职责

无额外接口，复用搜索接口和日常模式 API。

### 7.4 接口

前端搜索时并行调用：

1. `GET /api/v1/knowledge/search` → 文件列表
2. `POST /api/v1/workstation/chat/completions` → AI Overview（`knowledge_space_ids` 为全部门户绑定的知识空间）

搜索接口参数：

| `q` | `string` | 是 | 关键词 |

| `file_type` | `string` | 否 | 文档类型筛选 |
| `tag` | `string` | 否 | 标签筛选 |
| `sort` | `string` | 否 | `relevance`（默认）/ `update_time` |
| `page` | `int` | 否 | 默认 1 |

### 7.5 筛选选项来源

| 筛选项 | 来源 | 说明 |
| --- | --- | --- |

## 8. 模块三：业务域导航

### 8.1 功能描述

首页展示 13 个业务域（生产、投资、研发、采购、营销、财务、设备、安全、环保、质量、人力等），点击进入该域的文件列表页，支持按文件格式和标签筛选。

### 8.2 前端职责

- 业务域卡片/图标展示（前端写死 domain 名称、图标）
- 维护 `domain → space_id` 一对一映射
- 进入业务域后，并行调用三个接口获取文件列表和筛选选项
- 筛选交互（文件格式下拉框、标签下拉框）

### 8.3 后端职责

- 按 `space_id` 返回文件列表，支持 `file_type` 和 `tag` 筛选
- 返回空间下所有标签（复用现有接口）
### 8.4 接口

前端进入业务域页面时并行调用：

**1. 文件列表**

```
GET /api/v1/knowledge/space/{space_id}/files?file_type=pdf&tag=热轧&page=1&page_size=20
```

| 参数 | 位置 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- | --- |
| `space_id` | path | `int` | 是 | 知识空间 ID |
| `file_type` | query | `string` | 否 | 文件格式筛选 |
| `tag` | query | `string` | 否 | 标签筛选 |
| `page` | query | `int` | 否 | 默认 1 |
| `page_size` | query | `int` | 否 | 默认 20，最大 100 |

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [KnowledgeFileItem],
    "total": 42,
    "page": 1,
    "page_size": 20
  }
}
```

查询约束：

1. `knowledge_id = space_id`
2. `file_type = FILE`（数据库层面，区分文件和文件夹）
3. 仅返回 `status = SUCCESS`
4. 如果传了 `file_type` 参数，按文件名后缀过滤
5. 如果传了 `tag` 参数，通过 `TagLink` 过滤
6. 按 `update_time` 倒序

**2. 标签列表（复用现有）**

```
GET /api/v1/knowledge/space/{space_id}/tag
```

响应：Tag 对象数组，前端用 `name` 字段渲染筛选下拉框。

**3. 文件格式选项**

前端写死，不需要接口。常见格式：`pdf`、`docx`、`doc`、`xlsx`、`xls`、`pptx`、`ppt`、`txt`、`html`。

**4. 排序**

固定按 `update_time` 倒序（最近更新优先），不做排序选项。

## 9. 模块四/五/六：推荐内容、专题内容、知识广场

### 9.1 功能描述

首页按标签聚类展示不同分区的内容，包括：

- 知识推荐·最新精选（如标签 `精选`）
- 典型案例·事故分析（如标签 `典型案例`、`事故分析`）
- 其他专题栏目

### 9.2 统一实现方案

三个模块统一用**标签聚类**控制：

- 后台配置每个分区对应的标签名
- 首页每个分区调 `GET /knowledge/files?tag=xxx&page_size=4` 取最新 4 条展示
- 点击"更多" → 跳到通用列表页（复用搜索结果页样式，去掉 AI Overview）
- 点击卡片 → 跳到详情页

### 9.3 接口

新增跨空间按标签查文件接口：

```
GET /api/v1/knowledge/files?tag=典型案例&page=1&page_size=10
```

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `tag` | `string` | 是 | 标签名称 |
| `page` | `int` | 否 | 默认 1 |
| `page_size` | `int` | 否 | 默认 20，最大 100 |

实现：

- 不走 ES，直接 SQL 查 `TagLink` → `KnowledgeFile`
- 只查公开空间（`is_released=True` / `auth_type=public`）的文件
- 按 `update_time` 倒序
- 返回 `KnowledgeFileItem` 列表

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [KnowledgeFileItem],
    "total": 15,
    "page": 1,
    "page_size": 20
  }
}
```

### 9.4 知识广场说明

知识广场 = 全部配置到门户里的知识空间。展示所有已绑定的空间入口。

业务域导航 = 置顶板块，在配置后台选择哪些空间置顶展示在首页业务域区域。

两者的关系：

- 知识广场包含所有已配置的知识空间
- 业务域是从知识广场中选出的置顶空间，在首页优先展示

### 9.5 通用列表页

点击"更多"后进入的列表页复用搜索结果页样式：

- 分类标题 + 筛选栏（业务域、文档类型、标签） + 文件列表 + 分页
- **去掉 AI Overview**
- 前端根据入口判断是否显示 AI Overview

标题来源：

| 入口 | 标题 | 来源 |
| --- | --- | --- |
| 业务域 | 如"设备" | 前端配置的 domain 名称 |
| 推荐更多 | 如"知识推荐·最新精选" | 前端配置的分区标题 |
| 专题更多 | 如"典型案例·事故分析" | 前端配置的分区标题 |
| 知识广场 | 如"轧线技术案例库" | 知识空间名称（`Knowledge.name`） |

### 9.6 页面模板总结

全站只需两个列表页模板：

| 模板 | 组成 | 使用场景 |
| --- | --- | --- |
| 搜索列表页 | AI Overview + 筛选 + 文件列表 | 搜索结果 |
| 通用列表页 | 分类标题 + 筛选 + 文件列表 | 业务域、推荐更多、专题更多、知识广场 |

### 12.3 后端职责

- 返回文件元信息（新增接口）
- 返回文件预览 URL（复用现有接口）
- 返回基于标签的相关推荐（新增接口）

#### 12.4.2 文件预览（复用现有）

```text
GET /api/v1/knowledge/space/{space_id}/files/{file_id}/preview
```

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "original_url": "原始文件下载链接",
    "preview_url": "预览文件链接"
  }
}
```

前端使用 `preview_url` 渲染文件正文内容。

**2. 相关推荐**

```text
GET /api/v1/knowledge/space/{space_id}/files/{file_id}/related?limit=3
```

| 参数 | 位置 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- | --- |
| `space_id` | path | `int` | 是 | 当前文件所属空间 |
| `file_id` | path | `int` | 是 | 当前文件 ID |
| `limit` | query | `int` | 否 | 默认 3，最大 3 |

查询规则：同空间内按 `update_time` 排序，优先取前 1 篇 + 后 2 篇，排除自身，总数补齐 3 篇。前面不够则多取后面，后面不够则多取前面。

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [KnowledgeFileItem],
    "total": 3
  }
}
```

## 16. 门户后台配置管理

### 16.1 功能描述

门户需要一个后台配置页面，用于管理业务域与知识空间的绑定关系。

### 16.2 配置内容

| 配置项 | 说明 |
| --- | --- |
| 知识空间绑定 | 将 BiSheng 知识空间配置到门户中，构成知识广场的全量空间 |
| 业务域（置顶板块） | 从已绑定的空间中选择置顶展示的空间，配置业务域名称和图标 |
| 绑定的 space_id | 业务域与知识空间一对一绑定 |
| 绑定时缓存的 tags | 绑定时自动调用 `GET /knowledge/space/{space_id}/tag` 查出该空间全量标签并缓存 |
| 首页分区标签 | 配置首页各推荐/专题分区对应的标签名（如 `精选`、`典型案例`、`事故分析`） |
| 日常模式 knowledge_space_ids | 技术问答 / AI Overview 调用 `chat/completions` 时传入的知识空间范围，后台可配置 |

### 16.3 使用场景

所有模块的知识空间范围都从这一份配置出发：

- 业务域页面的标签筛选下拉框 → 使用该 domain 绑定时缓存的 tags
- 搜索结果页的标签筛选下拉框 → 使用所有 domain 缓存的 tags 合集
- 搜索结果页的业务域筛选下拉框 → 使用所有已配置的 domain 列表
- 业务域页面文件列表 → 使用绑定的 `space_id` 调用 `GET /knowledge/space/{space_id}/files`
- 全局搜索 → 搜索范围为全部已绑定的 `space_id`，传入 `GET /knowledge/search?space_id=`
- AI Overview → 调用日常模式 API，传入 `use_knowledge_base.knowledge_space_ids`
- 技术问答 → 调用日常模式 API，传入后台配置的 `use_knowledge_base.knowledge_space_ids`
